# Neutron Future Enhancements Roadmap

This document outlines potential improvements and features for future development of the Neutron programming language.

## ‚úÖ Completed Features

### 1. Break and Continue Statements ‚úÖ IMPLEMENTED
**Status:** Complete  
**Impact:** High  
**Effort:** Medium  
**Description:** Implement `break` and `continue` statements for loop control.

**Example:**
```neutron
for (var i = 0; i < 100; i = i + 1) {
    if (i == 50) {
        break;  // Exit loop
    }
    if (i % 2 == 0) {
        continue;  // Skip to next iteration
    }
    say(i);
}
```

**Implementation:**
- ‚úÖ Added `BREAK` and `CONTINUE` tokens
- ‚úÖ Added bytecode operations `OP_BREAK` and `OP_CONTINUE`
- ‚úÖ Track loop depth in compiler with breakTargets and continueTargets
- ‚úÖ Special handling for for-loops: continue jumps to increment before condition
- ‚úÖ Handle break/continue with jump instructions and loop stack management

**Test File:** `test_all_roadmap_features.nt`

### 2. Command Line Arguments ‚úÖ IMPLEMENTED
**Status:** Complete  
**Impact:** High  
**Effort:** Low  
**Description:** Pass command line arguments to scripts via `sys.args()`.

**Example:**
```neutron

```

**Implementation:**
- ‚úÖ Modified `main.cpp` to pass argc/argv to VM via `vm.setCommandLineArgs()`
- ‚úÖ Store arguments in VM as `std::vector<std::string> commandLineArgs`
- ‚úÖ Updated `sys_args()` in sys module to return actual arguments as array
- ‚úÖ Arguments include script name as first element

**Test File:** `test_all_roadmap_features.nt`

### 4. String Interpolation ‚úÖ IMPLEMENTED
**Status:** Complete  
**Impact:** High  
**Effort:** Medium  
**Description:** Allow embedding expressions directly in strings.

**Example:**
```neutron
var name = "World";

```

**Implementation:**
- ‚úÖ Updated scanner to recognize `${...}` in strings
- ‚úÖ Parse embedded expressions with chunk-based approach
- ‚úÖ Generate bytecode to concatenate parts using PLUS operations
- ‚úÖ Supports variables, literals, and complex expressions

**Test File:** `test_all_roadmap_features.nt`

### 5. Enhanced Array Operations ‚úÖ IMPLEMENTED
**Status:** Complete  
**Impact:** Medium  
**Effort:** Medium  
**Description:** Add comprehensive array manipulation methods.

**Implemented Features:**
```neutron
var arr = [1, 2, 3, 4, 5];

// Property:
arr.length;             // 5

// Basic methods:
arr.slice(1, 3);        // [2, 3]
arr.indexOf(3);         // 2
arr.join(", ");         // "1, 2, 3, 4, 5"
arr.reverse();          // [5, 4, 3, 2, 1]
arr.sort();             // [1, 2, 3, 4, 5]
arr.push(6);            // Add element
arr.pop();              // Remove and return last element

// Advanced methods with callbacks:
arr.map(fun(x) { return x * 2; });      // [2, 4, 6, 8, 10]
arr.filter(fun(x) { return x > 2; });   // [3, 4, 5]
arr.find(fun(x) { return x == 3; });    // 3
```

**Implementation:**
- ‚úÖ Added `length` property access for arrays in OP_GET_PROPERTY
- ‚úÖ Implemented all basic array methods in `callArrayMethod()`
- ‚úÖ Implemented advanced methods (map, filter, find) with callback support
- ‚úÖ **Critical VM Fix:** Modified `VM::run()` to accept `minFrameDepth` parameter
  - Enables nested execution contexts for callbacks from native code
  - Allows user functions called from C++ array methods to execute properly
- ‚úÖ Fixed stack management in array method calls

**Test Files:** `test_array_advanced.nt`, `test_all_roadmap_features.nt`
